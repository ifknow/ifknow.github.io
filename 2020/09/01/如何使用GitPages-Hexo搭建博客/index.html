如何使用GitPages+Hexo搭建博客

本文最后更新于：星期一, 八月 31日 2020, 8:26 晚上

快来康康如何使用GitHub+Hexo搭建属于自己的站点吧！

0 前言

现在博客托管网站也有很多了，我们可以在上面分享自己的生活和对知识的看法，比如CSDN、博客园、掘金等等。个人也就使用过CSDN和掘金，但是使用过一段时间后，就会发现这些博客网站或多或少，不太方便。

所以来说，拥有一个自己的博客站点，是非常有必要的了。

在网上搜索资料，发现目前比较流行的就是使用 GitPages + Hexo 来搭建一个静态博客站点。但是一直没找到合适的时间来倒腾。正好这周末休息两天，可以有时间搞搞了。经过筛选我发现一款非常好看、简介的 Hexo主题 - Fluid 。先贴一张图，看看效果：

网上有很多的建站教程，这里我就记录一下我搭建 GitPages + Hexo 的过程和在建站过程中遇到的问题，做一个记录。

点击👉查看Fluid 博客样式 www.gongshiyong.online ，不喜欢可以自己选择其他主题.

1 准备工作

1.1 配置环境

Win10

Git ：Git安装教程参考
Nodejs：Nodejs下载地址

Node.js 安装完成可以设置镜像加速，如果不加速的话，国内下载会超级慢！打开cmd。

npm config set registry https://registry.npm.taobao.org

注意：Git 和 Nodejs 是必须要安装的，不然没办法下面的操作！

1.2 注册GitHub账号

我们要把我们的博客放在 GitHub 上托管，国内访问 GitHub 也是很慢。（你可以使用插件或者代理加速GitHub访问）

有了账号之后新建一个仓库，命名为用户名.github.io。


这样你就创建了一个 可以以 用户名.github.io访问的网站了。直接在地址栏中输入，就可以打开，难免会有些慢，后面会介绍我是如何加速的。现在页面只会显示你的readme页面的内容。

继续配置吧，说远了。
点击仓库名进入仓库，点击 Setting 进入仓库设置页面，在 Setting 页面下拉到 最下面的 GitHub Pages 设置项。

到现在为止 博客存放的仓库设置好了。

2 安装 Hexo

在电脑本地，任意选择一个位置，创建一个文件夹，用来存放本地博客文件。进入该文件夹内，右键选择 Git Bash Here（安装完 Git 就会出现）。如果你熟悉 Linux 环境的话，Git 的命令控制台就会 很容易，命令都是通用的。

  • 输入下面命令安装 Hexo
npm install hexo-cli -g
  • 安装完成后输入下面命令来检验 Hexo 安装的版本
hexo -v

这样就是安装成功了，应该不会安装不成功的。

  • 初始化 Hexo 网站

hexo init

这个命令就是在该文件夹下初始化一个 Hexo 网站，这个过程会比较慢，也没有任何输出，等待就行。

  • 安装所需依赖

npm install

这样本地的 hexo 就创建完成了，输入下面两个命令本地预览 http://localhost:4000/。

hexo g # 编译md文件成为html页面
hexo s # 启动服务器

停止 运行快捷键 Ctrl + C ！

3 本地博客推送到 GitHub

本地博客可以预览之后，就可以推送到 GitHub 的刚创建的 仓库进行访问了。

这里我就默认你的Git环境已经配置ok了，有问题，参考☞ Git安装教程

打开 hexo 安装根目录，用编辑器打开 _config.yml，修改最后的 deploy配置。下面是我的配置，请按需修改。

deploy:
  type: git
  repository: https://github.com/ifknow/ifknow.github.io # 你的仓库地址
  branch: master # 选择默认master分支

  • 创建一个Test文章

hexo new Test
$ hexo new Test
INFO  Validating config
INFO  Created: C:\opensource\blog-post\hexo\source\_posts\Test.md

然后就会在source/_post/下面生成一个Test.md文件，随便在该文件中输入一些内容，保存。

这就是以后写文章的步骤！hexo new '文章名'。

  • 推送

hexo g # 编译md文件->html
hexo d # 推送

推送时遇到的问题？

  • 1、有时莫名其妙会让你输入用户名和密码，这里的用户名和密码是你的 GitHub 登录名和密码。
  • 2、github push failed (remote: Permission to userA/repo.git denied to userB？
    • 原因：由于你在本地配置 Git已经指定了两个用户。
    • 解决方法

现在你就可以使用 youname.github.io进行访问了。

4 域名绑定

Gitpages 支持自定义域名。前提是 你得一个域名，域名购买也很简单，备案也不是很复杂。阿里云或者其他平台都可以。

  • 1、 gitHub -> Setting -> GitHub Pages 中修改 ustom domain 填写你自己的域名，示例：gongshiyong.online。
  • 2、 这里我使用的阿里云域名，进入阿里云域名解析管理平台，其他的类似。添加标注的两条解析记录。

5 主题设置

5.1 下载主题

git clone https://github.com/fluid-dev/hexo-theme-fluid.git

完成之后在/themes/目录下出现一个hexo-theme-fluid文件夹。将该文件夹重命名为fluid。
点击查看 ☞ Fluid主题配置官文档

5.2 切换主题

打开根目录下的_config_yml。检索theme关键字，修改theme的值，为fluid。

hexo g
hexo d

主题切换成功。详细的主题配置，参考官网文档。

到此，使用 GitPages + Hexo 就可以通过自定义域名访问了。

但是由于国内访问 Git 过于缓慢，对于博客访问不太友好。访问太慢有时就会造成 连接超时的问题。经过各个方面考虑，我决定使用阿里云的 CDN 加速。

    选择左侧【域名管理】-> 添加域名。
  • 下一步。方法1：在域名解析中添加如下解析。方法2不做演示了。
  • 返回域名管理出现这个就是成功了。

  • 点击【监控查询】【用量查询】【资源包】，点击【购买资源包】，购买即可。
  • 之后写博客就只需要 打开 Git Base 命令框 输入 下面命令就可以新增一篇文章了
hexo new '文章名' # 文章名不能有空格，创建一篇文章后，就可以使用 markdown 编辑器 写博客
