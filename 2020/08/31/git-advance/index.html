Git（二）分支创建合并

本文最后更新于：2020-09-08 星期二 06:09:30 晚上

Git进阶-分支学习！

之前我们了解过Git的安装和简单操作。参考：Git（一）简介和入门使用 。
在现实的开发场景中，Git 的使用不仅仅只有一个 master分支，团队成员每个人都会有一个分支，自己将自己写的代码提交到自己的分支上，最后由一个人将其他分支逐一合并到 主分支上。

为保证文章的完整性，文章简单介绍下 Git 的基础使用

安装就略过…

1、准备工作

  • 在 GitHub/Gitee上面 创建一个仓库
  • 本地 Git 安装成功

2、拉取项目

将远程 GitHub/Gitee 上的项目 clone 到本地。

git clone https://github.com/ifknow/spring-boot-examples.git

新创建的仓库只有一个 master分支。

使用 集成开发工具 IDEA 打开项目，打开 IDEA Terminal / cmd / Git bash

  • 1、使用命令查看所有分支
git branch

目前只有一个master分支

  • 2、创建分支
git branch dev-test1
  • 3、切换分支（当前分支前会有一个 *）
git checkout dev-test1
# 快速创建并切换分支
git checkout -b dev-test2

3、分支操作

  • 1、分支提交

上面创建好的分支，使用下面的命令将新分支推送到远程仓库。

git push origin dev-test1

注意：向远程仓库推送时，第一次提示输入用户名密码，这里的用户名密码是你的 远程 GitHub / Gitee 的登录名和对应的密码。

这里我以 dev-ifknow``master两个分支进行演示分支合并。

我已经在 分支 dev-ifknow提交了最新的代码，将代码合并到 master分支。

合并前需要切换到 master分支

git merge dev-ifknow

使用merge命令将dev-ifknow的代码同步到本地master分支

git push origin master

使用上面的命令将本地最新代码推送到远程 master分支。

到此，Git 分支的创建和合并就到这里了。

4、删除分支

将刚才的创建的测试分支进行删除并推送到远程

git push origin -d dev-test1

Git 的使用可能会出现冲突等错误，感兴趣的可以自己研究。
